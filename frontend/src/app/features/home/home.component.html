Why feature-first?

Organize by domain, not by artifact

Each folder under features/ owns its routes, UI, and logic. The architecture stays discoverable as the product grows, without long import chains.

Folder layout

Clear separation of concerns

core/

Hosts app-wide layout, configuration, and interceptors. Should stay small and stable.

features/

Domain slices with self-contained entry routes, components, and data access.

shared/

Reusable UI elements and utilities that remain framework-agnostic.
